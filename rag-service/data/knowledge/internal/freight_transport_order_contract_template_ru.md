# Заявка на перевозку и смежные документы (черновик для RAG)

**Назначение:** какие поля обычно нужны для **заявки на перевозку** и для связки с **договором перевозки** (перечень данных, не юридическая подстановка типовой формы ГК).

**Не является:** утверждённой формой, официальным бланком ТТН/CMR или заменой юридической проверки.

**Макет PDF в продукте:** по умолчанию `pdf_template: "dogovor_zayavka"` — структура как у типового **договора-заявки на разовую автоперевозку**: шапка с № и датой, преамбула «Заказчик» / «Перевозчик», таблица условий (маршрут, груз, адреса, ТС, оплата), типовые пункты об обязанностях/спорах/ответственности, блок реквизитов и подписей. Упрощённый вид — `pdf_template: "simple"`.

**Ollama / LLM не печатает форму:** модель только заполняет **текстовые поля** (JSON в `POST /freight/transport-order-compose`). Бинарный **PDF** строит **бэкенд** (`POST /freight/transport-order-pdf`): по умолчанию встроенный движок **fpdf**, либо **`pdf_engine: "libreoffice"`** — тогда сервис пишет временный HTML и вызывает **LibreOffice** (`soffice --headless --convert-to pdf`) на той же машине, где запущен API (переменная `LIBREOFFICE_SOFFICE_PATH` при необходимости). Ollama к LibreOffice не подключается.

## Заявка на перевозку (типовое содержание)

1. **Заказчик** — наименование, адрес, контакт, телефон; при необходимости ИНН/КПП.  
2. **Перевозчик** — если уже известен; иначе блок можно оставить пустым или с формулировкой «подлежит согласованию».  
3. **Погрузка** — адрес или объект, желаемые дата и время (или окно).  
4. **Выгрузка** — адрес или объект, дата/время.  
5. **Груз** — наименование, масса (т/кг), объём (м³), количество мест, упаковка.  
6. **Транспорт** — тип кузова (тент, реф, изотерм и т.д.), особые требования (ADR, негабарит — только если это в фактах).  
7. **Коммерческие условия** — ставка или порядок расчёта, если стороны уже обсудили; иначе «по согласованию».  
8. **Дополнительно** — страхование, пломбы, скоропорт, ответственность — по фактам из переписки/договора.

---

## Связь с договором перевозки

- Заявка часто **предшествует** или **дополняет** договор: в договоре фиксируют общие условия, в заявке — конкретный рейс (маршрут, даты, параметры груза).  
- Номер и дата договора полезно указывать в шапке или в преамбуле заявки, если они известны.  
- Полноценный **текст договора** готовит юрист по шаблону компании; RAG даёт только **структуру полей** и черновик заявки.

---

## Правила для модели при заполнении полей

- Переносить в поля только то, что есть в запросе пользователя; не придумывать ИНН, адреса и суммы.  
- Пустые обязательные для отправки реквизиты — вынести в `missing_information` (для API compose).  
- Для выгрузки в **PDF** используйте endpoint `POST /freight/transport-order-pdf` с JSON полей; шрифт и вёрстка — на стороне сервиса.
