## Итоговый счёт (прогон B, целевые критерии)

| Критерий | Результат |
|-----------|-----------|
| Кейсы **3** и **4** остаются защитными / дисциплиной | **3** — **ok**; **4** — **ok** по форме |
| **1–2** без multiple ranges | **Да** в прогоне B |
| **1–2** без смешения «ориентир/ставка» в виде двух заголовков подряд | **Улучшено** в B; кейс **2** всё ещё семантически «ориентир» при запросе «рыночная ставка» |
| **5** — явная маркировка необоснованности | **Да** в structured fields (прогон B) |

Ориентировочно: **~3 ok**, **~2 partial** (1 по фактам, 2 по смыслу «ставка»), **0 fail** по жёстким чекпоинтам «несколько вилок» после B.

Цель **4 ok / 1 partial** — почти достигнута; остаётся дожать **кейс 2** (согласованность «достаточно данных → режим ставка» без ложного «данных мало») — вероятно **prompt + качество тарифных чанков / post-retrieval normalization**, как ты и писал.
