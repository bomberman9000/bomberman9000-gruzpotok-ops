## Summary

| Метрика | Значение |
|---------|----------|
| **ok** | **2** (кейсы 3, 4) |
| **partial** | **3** (кейсы 1, 2, 5) |
| **fail** | **0** |

**Чаще всего проседает слой:** **prompt** (дисциплина одного режима, краткость, согласованность цифр и текста; в кейсе 2 дополнительно возможный вклад **retrieval**/качества знаний под тарифы).

**Следующий наиболее вероятный TOP ISSUE (после появления разметки операторов):**

1. **`too_generic`** / избыточная многословность и «двойные» ответы (кейс 2).  
2. Повторение **`bad_price_range`** или схожих тегов, если операторы отметят **нереалистичные или широкие вилки** (кейсы 1–2).  
3. **`insufficient_context`** в **legal**, если ждать явного разбора необоснованной суммы в претензии (кейс 5).

**Review-gap:** в экспорте нет `operator_action` / `reason_codes` — до ревью в UI **`analyze_session` по причинам пустой**; для метрик нужна разметка по этим пяти кейсам в продукте.
