# Работа без стабильного внешнего интернета

Цель: чтобы разработка и прогон стека не зависели от постоянного доступа в интернет. Это обычная инженерная подготовка (как для авиарейсов, так и для ограничений канала).

## Что в этом репозитории уже «локальное»

По `docker-compose.yml` типовой контур:

- **PostgreSQL + pgvector** — данные и эмбеддинги у вас.
- **Redis** — кэш/очереди локально.
- **rag-api** — читает знания из `rag-service/data/knowledge`, вызывает **Ollama на хосте** (`OLLAMA_BASE_URL`, чаще `http://host.docker.internal:11434`).
- **Backend** ходит в **rag-api по LAN** (`RAG_API_BASE_URL`), не в OpenAI по умолчанию.

То есть **инференс и RAG** могут работать в изолированной сети, если заранее подняты контейнеры, БД и модели Ollama.

## Что всё ещё требует интернет (один раз или периодически)

| Задача | Зачем кэшировать |
|--------|------------------|
| `pip install` / `uv sync` | Зависимости `backend/requirements.txt`, `rag-service/requirements.txt` |
| `docker pull` / `docker compose build` | Базовые образы (`postgres`, `redis`, сборки `Dockerfile`) |
| `ollama pull …` | Модели чата и эмбеддингов (`OLLAMA_MODEL`, `EMBEDDING_MODEL` и т.д.) |
| `git fetch` / клонирование | Обновления кода |

Имеет смысл **заранее** (пока канал есть) собрать артефакты на отдельный диск или внутренний сервер.

## Практические шаги подготовки

1. **Python-зависимости**  
   На машине с доступом: установить в venv и сохранить дистрибутивы, либо использовать корпоративный **PyPI-mirror** / **devpi** / артефакт-хранилище. Потом ставить пакеты с внутреннего индекса или из каталога `.whl`/`.tar.gz`.

2. **Docker-образы**  
   `docker pull` нужных тегов, затем `docker save` в файлы и перенос на офлайн-хост; там `docker load`. Образы из compose перечислены в `docker-compose.yml` (postgres, redis, сборки из `./rag-service`, `./backend`).

3. **Ollama**  
   Заранее `ollama pull` для моделей из `.env` / compose. Модели лежат в пользовательском каталоге Ollama на хосте — при необходимости переносятся вместе с бэкапом диска (см. документацию Ollama по путям для вашей ОС).

4. **Данные знаний**  
   Каталог `rag-service/data/knowledge` уже в репозитории; дополнительные материалы — под версионирование или бэкап.

5. **Документация и исходники**  
   Клон репозитория + архив релиза; при полном офлайне — локальный просмотр Markdown в IDE.

## Внутренний API и журнал eval

`export_recent_cases.py` обращается к **`API_BASE`** (часто `http://127.0.0.1:8090`). Для офлайна достаточно **локального backend + БД** с тем же `DATABASE_URL`. Журнал `docs/evals/session_log.json` можно вести полностью офлайн.

## Что не входит в этот документ

Настройки маршрутизации, VPN и обходы фильтрации — это зона **корпоративной ИБ и законодательства**; здесь только схема «локальный стек + заранее скачанные артефакты».
